# Especificación de Requisitos - SimRacing Shop

## Alcance

Este documento especifica los requisitos funcionales y no funcionales del MVP de SimRacing Shop, una plataforma e-commerce para productos de sim racing personalizables con configurador 3D. 

Utilizando estructuras como:
- **Who - What - Why** para historias de usuario
- **Given - When - Then** para los criterios de aceptación

Técnicas y principios como **MoSCoW** e **INVEST** así como objetivos medibles.

---

## Leyenda de Prioridades (MoSCoW)

| Prioridad | Significado | Descripción |
|-----------|-------------|-------------|
| **Must** | Debe tener | Requisito crítico, bloqueante para el MVP |
| **Should** | Debería tener | Importante para el MVP pero no bloqueante |
| **Could** | Podría tener | Deseable si hay tiempo y recursos |
| **Won't** | No tendrá (ahora) | Fuera del alcance del MVP actual |

# PARTE 1: REQUISITOS FUNCIONALES

## 1. Autenticación y Usuarios

### RF-001: Registro de Usuario

**ID:** RF-001
**Prioridad:** Must
**Descripción:** Como visitante, quiero crear una cuenta en la plataforma, para poder realizar compras y guardar mis configuraciones personalizadas.

**Criterios de Aceptación:**

1. **Dado que** soy un visitante en la página de registro, **cuando** completo el formulario con email válido, contraseña segura y acepto los términos, **entonces** mi cuenta se crea y recibo confirmación.
2. **Dado que** intento registrarme con un email ya existente, **cuando** envío el formulario, **entonces** veo un mensaje de error indicando que el email ya está registrado.
3. **Dado que** ingreso una contraseña con menos de 8 caracteres, **cuando** intento enviar el formulario, **entonces** veo un mensaje indicando los requisitos mínimos de contraseña.
4. **Dado que** no acepto los términos y condiciones, **cuando** intento enviar el formulario, **entonces** el envío se bloquea y se indica que es obligatorio.
5. **Dado que** el CAPTCHA no se completa correctamente, **cuando** intento enviar el formulario, **entonces** el registro se rechaza para prevenir bots.

**Campos del Formulario:**
- Email (obligatorio)
- Contraseña (obligatorio, min 8 chars, 1 mayúscula, 1 número)
- Confirmar contraseña (obligatorio)
- Nombre (opcional)
- Apellidos (opcional)
- Checkbox términos y condiciones (obligatorio)

---

### RF-002: Login de Usuario

**ID:** RF-002
**Prioridad:** Must
**Descripción:** Como usuario registrado, quiero iniciar sesión con mis credenciales, para acceder a mi cuenta y realizar compras.

**Criterios de Aceptación:**

1. **Dado que** tengo una cuenta válida, **cuando** ingreso email y contraseña correctos, **entonces** inicio sesión y soy redirigido a mi destino.
2. **Dado que** ingreso credenciales incorrectas, **cuando** envío el formulario, **entonces** veo un mensaje genérico de error sin revelar qué campo falló.
3. **Dado que** marco la opción "Recordarme", **cuando** inicio sesión, **entonces** mi sesión persiste mediante refresh token.
4. **Dado que** olvidé mi contraseña, **cuando** hago clic en "¿Olvidaste tu contraseña?", **entonces** soy redirigido al flujo de recuperación.

---

### RF-003: Recuperación de Contraseña

**ID:** RF-003
**Prioridad:** Must
**Descripción:** Como usuario registrado, quiero poder recuperar mi contraseña si la olvido, para no perder acceso a mi cuenta.

**Criterios de Aceptación:**

1. **Dado que** solicito recuperación con un email registrado, **cuando** envío el formulario, **entonces** recibo un email con link de reset válido por 1 hora.
2. **Dado que** tengo un link de reset válido, **cuando** establezco una nueva contraseña que cumple los requisitos, **entonces** mi contraseña se actualiza y recibo confirmación por email.
3. **Dado que** intento usar un link de reset expirado, **cuando** accedo al link, **entonces** veo un mensaje indicando que el link ha expirado.
4. **Dado que** solicito recuperación con email no registrado, **cuando** envío el formulario, **entonces** veo el mismo mensaje de éxito (por seguridad) pero no se envía email.

**Umbrales envío gratis:**
- Península: > 100€
- Baleares: > 150€
- Canarias: > 200€

### RF-028: Integración con Stripe

**ID:** RF-028
**Prioridad:** Must
**Descripción:** Como sistema, necesito procesar pagos de forma segura con Stripe, para completar transacciones de compra.

**Criterios de Aceptación:**

1. **Dado que** el usuario confirma el pago, **cuando** se procesa, **entonces** se crea un PaymentIntent en backend.
2. **Dado que** la tarjeta requiere 3D Secure, **cuando** se detecta, **entonces** Stripe maneja la autenticación SCA.
3. **Dado que** el pago es exitoso, **cuando** Stripe notifica via webhook, **entonces** el pedido se actualiza a "Paid".
